# Self 规则

`self/` 是 second me 的种子。它不是日记，也不是所有行为流水账，而是压缩、可溯源、谨慎的用户表示。

## 五件套

- `self/observations.md`：原子观察，事实源，只追加。
- `self/identity.md`：稳定身份、价值观、原则、思维方式、决策依据。
- `self/preferences.md`：可操作偏好，按适用场景组织。
- `self/now.md`：当前项目、近期目标、活跃问题、当前约束。
- `self/profile.md`：500 token 以内的 second me 压缩出口。

## 证据等级

- 明确事实：用户直接说出。默认可信，但仍记录来源和日期。
- 强推断：多次行为或多个来源共同支持。
- 弱信号：一次性行为、主题兴趣、单次收藏、单次蒸馏方向。

明显 self 信号可以写入 observations，并按需要更新 identity/preferences/now。弱信号不要当场升级；让它留在 `log.md`，由审查模式聚合。

## 抽取维度

- 身份背景：职业、教育、角色、经历、所在地。
- 价值观与原则：在乎什么，反感什么，如何取舍。
- 思维方式：怎么推理、怎么决策、容易卡在哪里。
- 沟通偏好：简洁/详尽、语气、反馈方式。
- 工作/学习偏好：怎么学、怎么做项目、怎么组织信息。
- 审美偏好：反复被什么视觉、设计、语言风格吸引。
- 当前关注：正在做什么，近期目标是什么。

## 写入规则

- 先写 observations，再从 observations 归并到 identity/preferences/now。
- 每条 observation 只说一件事，带日期、证据等级、来源。
- identity 只收稳定内容，不收短期事项。
- preferences 必须说明适用场景，避免无差别套用。
- now 要定期更新，旧事项移到历史区，不直接删除。
- profile 只放稳定且对未来回答有用的信息，控制在 500 token 以内。

## 敏感信息

不要从弱证据推断受保护特征、医疗状况、法律状态、财务状况或私人关系。身份定义类、性格判断类、高影响结论，证据模糊时先写入 `nextstep.md` 等用户确认。
